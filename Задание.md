# 2. Задание: Financial Ledger API

## 2.1 Бизнес-контекст

Вам необходимо реализовать упрощённую систему финансового учёта (Financial Ledger), работающую по принципам двойной бухгалтерии.

---
## 2.1.1 Основные сущности

### Account (Счёт)

Представляет финансовый счёт, например:
- Cash (Наличные)
- Revenue (Доходы)
- Expenses (Расходы)
- Liability (Обязательства)

Каждый счёт имеет:
- id (рекомендуется UUID)
- name (уникальное)
- type (ASSET, LIABILITY, REVENUE, EXPENSE)

⚠ Баланс не должен храниться как изменяемое поле — он вычисляется на основе транзакций.

---

### Transaction (Транзакция)

Финансовая операция, затрагивающая один или несколько счетов.

Каждая транзакция содержит:
- id
- timestamp
- description
- коллекцию записей (entries)

---
### Transaction Entry (Проводка)

Представляет дебет или кредит внутри транзакции.

Каждая запись содержит:
- account_id
- type (DEBIT или CREDIT)
- amount (положительное decimal число)

---
## 2.1.2 Правило двойной бухгалтерии

Критическое бизнес-правило:

В каждой транзакции сумма дебетов должна равняться сумме кредитов.

Пример:  
Покупка канцтоваров на $100 за наличные:
- DEBIT: Supplies +100
- CREDIT: Cash -100

Сумма дебетов = 100  
Сумма кредитов = 100

Это правило должно проверяться на уровне бизнес-логики.

---
# 2.2 Требуемый функционал

## 2.2.1 Управление счетами

### Создание счёта

POST /api/accounts

Request:

{  
  "name": "Cash",  
  "type": "ASSET"  
}

Response:  
201 Created  
Детали счёта, включая вычисленный баланс (изначально 0).

---

### Получить все счета

GET /api/accounts

Возвращает список счетов с вычисленными балансами.

---

### Получить счёт по ID

GET /api/accounts/{id}

Возвращает:
- Детали счёта
- Текущий вычисленный баланс

---

## 2.2.2 Обработка транзакций

### Создать транзакцию

POST /api/transactions

Пример запроса:

{  
  "description": "Purchase office supplies",  
  "date": "2024-01-15T10:30:00",  
  "entries": [  
    {  
      "accountId": "uuid",  
      "type": "DEBIT",  
      "amount": 100.00  
    },  
    {  
      "accountId": "uuid",  
      "type": "CREDIT",  
      "amount": 100.00  
    }  
  ]  
}

Необходимо проверить:
- Минимум 2 записи
- Хотя бы один DEBIT и один CREDIT
- Все счета существуют
- Суммы положительные
- Сумма дебетов == сумме кредитов

Ответ:  
201 Created или ошибка валидации.

---

### Получить транзакцию по ID

GET /api/transactions/{id}

Возвращает транзакцию с проводками.

---

### Получить транзакции счёта

GET /api/accounts/{id}/transactions

Возвращает:
- Все транзакции, затрагивающие счёт
- Детали счёта
- Текущий вычисленный баланс

---

## 2.2.3 Расчёт баланса

Баланс рассчитывается динамически на основе проводок.

Типы счетов ведут себя по-разному:

|Тип счёта|Увеличение|Уменьшение|
|---|---|---|
|ASSET|DEBIT|CREDIT|
|LIABILITY|CREDIT|DEBIT|
|REVENUE|CREDIT|DEBIT|
|EXPENSE|DEBIT|CREDIT|

Баланс:

- Не хранится как изменяемое поле
- Может вычисляться:
    - В Python
    - Или SQL-агрегацией

---

# 2.3 Требования к валидации

Необходимо обеспечить:
1. Уникальность имени счёта
2. Имя не может быть пустым
3. Корректный enum типа счёта
4. Транзакция содержит ≥ 2 записей
5. Дебет = кредит
6. Счета существуют
7. Суммы положительные
8. Описание не пустое

HTTP-коды:
- 400 — ошибка валидации
- 404 — ресурс не найден
- 409 — нарушение уникальности

⚠ Бизнес-логика не должна находиться в API-слое.
